A tracker based on the OpenTLD library.

Description

This is a yarp wrapper of the TLD algorithm (a.k.a. Predator) performing Tracking, Learning and Detection.

Available RPC services are:

(notation: [.] identifies a vocab, <.> specifies a double, "." specifies a string)

LOAD
format #1: [load] "model.md"
reply: [nack]|[ack] "path_to_model.md"
action: load the model from the specified file.
format #2: [load] <top-left.x> <top-left.y> <bottom-right.x> <bottom-right.y>
action: init TLD with the template given by the bounding box.

SAVE
format: [save] "model.md"
reply: [nack]|[ack] "path_to_model.md"
action: store the model in the specified file.

START
format: [start]
action: start tracking/learning/detecting.

STOP
format: [stop]
action: stop tracking/learning/detecting.

CLEAR
format: [clear]
action: clear the content of TLD.

SET
format: [set] [learn]|[alternating] [on]|[off]
action: enable/disable learning | alternating mode.
In alternating mode, the detection is off while tracking.

GET
format: [get] [learn]|[alternating]
reply: [nack]|[ack] [on]|[off]
action: retrieve learning | alternating current status.

Libraries

  • YARP libraries.
  • OpenCV (with C++ layer).
  • OpenTLD.

Installing OpenTLD

The steps are:

  1. git clone https://github.com/gnebehay/OpenTLD.git .
  2. compile the library with default values of cmake variables.
  3. set up the environment variable OpenTLD_ROOT pointing to the root directory.
  4. set up the environment variable OpenTLD_DIR pointing to the build directory.
  5. For Windows systems only, when cmaking TLD project, select which sub path to the OpenTLD libraries is to be used, if Release (default) or Debug, by filling the OpenTLD_WIN_LIB_SUBPATH variable.

Ports Created

  • / <modName>/img:i receives the image acquired from the camera.
  • / <modName>/img:o streams out the image containing tracked object.
  • / <modName>/data:o streams out information about the tracked object in the following format: <top-left.x> <top-left.y> <bottom-right.x> <bottom-right.y> <confidence>.
  • / <modName>/rpc processes rpc requests.

Parameters

–name name

  • specify the module stem-name, which is tld by default. The stem-name is used as prefix for all open ports.

–context path

  • specify the path to the model files to be loaded/written.

–model file

  • specify the model file to load the TLD with at start-up.
